본 문서는 최근 개발 서버(oci2)와 운영 서버(oci) 간 무장애 배포를 목적으로 수행된 백엔드, 프론트엔드, 스크래퍼 시스템의 핵심 기능 업데이트 및 리팩토링 변경 사항을 기록합니다.

## 📅 최근 릴리즈 일자: 2026년 6월 7일

### 🌟 1. FnGuide 요약 리포트 - 증권사 원본 리포트(tbl_sec) 정밀 유사도 매칭 엔진 구축
*   **유사도 매칭 서비스 도입 (`FnGuideMatcher`):**
    *   수집된 FnGuide 요약 정보(`FnGuideReportSummary`)와 증권사 PDF 리포트(`SecReport`) 간의 데이터 유기적 매칭을 수행하는 매칭 엔진을 신규 설계 및 적용했습니다.
    *   회사명, 애널리스트명(작성자), 등록일 및 제목 유사도를 종합적으로 점수화하여 가장 정밀한 대상에 대해 바인딩(`fnguide_summary_id`)을 진행합니다.
*   **데이터 스키마 및 API 상호 연동 연동:**
    *   `/api/fnguide/report-summaries` API 등에서 매칭된 증권사 리포트의 실제 원본 PDF URL(`pdf_url`) 정보를 상호 즉각 추적할 수 있도록 연동 및 매핑 로직을 고도화했습니다.

### 🛡️ 2. 보안 헤더 기반 내부 매칭 API 도입 및 스케줄러 경로 장애 원인 제거
*   **환경 격리 및 경로 의존성 제거:**
    *   기존 스케줄러(크론탭) 환경에서 백엔드 로컬 가상환경 파이썬 바이너리를 직접 실행하며 발생하던 `No such file or directory` (Errno 2) 경로 장애 문제를 근본적으로 해소했습니다.
    *   스케줄러가 파이썬 환경을 직접 타지 않고 백엔드의 가벼운 HTTP REST API 호출을 통해 매칭 작업을 안전하게 위임·트리거하도록 구조를 재설계했습니다.
*   **내부 전용 API 구축 (`/admin/fnguide/match-internal`):**
    *   내부 시스템 전용 매칭 API를 신규 개설했습니다.
    *   외부 공격 위협으로부터 API를 통제하기 위해 `X-Internal-Token` 헤더 보안 인증 체계를 추가하였으며, 설정된 비밀 키(`jwt_secret_key`)와 불일치 시 `403 Forbidden`을 엄격히 반환하도록 구현했습니다.
*   **데이터베이스 영속성 및 철저한 보안 테스트 추가:**
    *   `tests/test_fnguide_matcher.py`에 `test_trigger_fnguide_match_internal_api` 검증 코드를 철저하게 추가했습니다.
    *   토큰 누락, 잘못된 토큰 시 예외 처리뿐 아니라, 실제로 DB 세션 초기화(`db_session.expire_all()`) 이후 매칭 결과가 실제 데이터베이스 레이어에 완전하게 커밋(Commit)되었는지를 빈틈없이 교차 검증합니다.

### 🤖 3. AI 다중 요약 엔진 (DeepSeek & Antigravity) 통합 및 다중 엔진 분기 대응
*   **Antigravity(Gemini REST) 요약 매니저 설계:**
    *   기존 DeepSeek 요약 모듈 외에, Antigravity(Gemini API) 요약 엔진을 병렬 탑재하기 위해 `AntigravityManager`를 신규 개발했습니다.
    *   `/admin/reports/{report_id}/summarize` API 엔드포인트에 쿼리 매개변수 `?engine=deepseek|ag`를 제공하여 요약 엔진을 자유롭고 안전하게 분기할 수 있도록 리팩토링했습니다.
*   **프론트엔드 관리자 UI 다중 요약 버튼 고도화:**
    *   관리자 콘솔에서 각 보고서별로 원하는 LLM 엔진을 사용하여 즉각 요약을 트리거할 수 있도록 UI 단을 전면 개선했습니다.
    *   **DeepSeek 요약 버튼:** 미래지향적 인공지능 정보망을 은유하는 블루 컬러 스타일 및 기하학적 SVG 아이콘 적용.
    *   **Antigravity 요약 버튼:** 반중력과 상승 포탈을 은유하는 퍼플 컬러 스타일 및 상승 화살표 기하학 SVG 아이콘 적용.
    *   UI 상에서 어떤 버튼이 마킹되었는지 직관적으로 알려주는 활성화(Active) 상태 디자인 및 요약 확인 팝업 인터랙션을 설계했습니다.

### ⚡ 4. 프론트엔드 네트워크 성능 최적화 및 중복 태그 정제
*   **API 중복 페칭(Fetching) 억제:**
    *   메인 리포트 상세 페이지 및 검색 화면에서 단일 동작으로 인해 과도한 중복 API 호출이 전송되는 비효율을 방지하도록 리소스를 제어하고 비동기 요청을 최적화했습니다.
*   **중복 렌더링 태그 필터링:**
    *   리포트 카드의 종목(Stock Names) 및 섹터(Sector) 태그 렌더링 시 간헐적으로 노출되던 중복 태그 노이즈를 완전 제거하여 컴포넌트의 렌더링 일관성을 보완했습니다.
